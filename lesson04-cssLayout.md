# CSS Layout

## display

### block
새 줄로 시작. 좌우 대칭. `div`, `p`, `form`, `header`, `footer`, `section`

### inline
`span`이 표준 인라인 엘리먼트. 단락 흐름을 방해하지 않고 텍스트를 감쌀 수 있다. `a`

### none
dom이 사라진거 처럼. `script`의 기본값임. `visibility:hidden`과 다름.

### 전체 목록

[MDN display](https://developer.mozilla.org/en-US/docs/Web/CSS/display)

## margin:auto;

```css
#main {
  width: 600px;
  margin: 0 auto;
}
```

block level은 `margin: auto;`를 하면 width만큼의 폭으로 중앙 정렬을 한다. **문제는** 브라우저 폭이 wdith보다 작을 때.


## max-wdith

```css
#main {
  max-width: 600px;
  margin: 0 auto;
}
```
브라우저 폭에 반응해서 **600이상 커지지 않는다.**로 이해하면 편함. **IE7+**지원

## Box model
오랬동안 width를 지정할 때, margin, padding을 고려해야 한다는게 문제였음.

```css
.simple {
  width: 500px;
  margin: 20px auto;
}

.fancy {
  width: 500px;
  margin: 20px auto;
  padding: 50px;
  border-width: 10px;
}
```

## box-sizing
[MDN box-sizing](https://developer.mozilla.org/en-US/docs/Web/CSS/box-sizing)에 자세한 내용이 있지만, 간단히 말해서 `boarder-box`로 지정하면 width가 고정됨. brower prefix가 필요함. **IE8+**

- `content-box`는 border, margin, padding을 포함하지 않음.
- `border-box`는 padding, border 포함한 margin은 포함하지 않음.
- `padding-box`는 padding을 포함함, border, margin은 포함하지 않음.

```css
.simple {
  width: 500px;
  margin: 20px auto;
  -webkit-box-sizing: border-box;
     -moz-box-sizing: border-box;
          box-sizing: border-box;
}

.fancy {
  width: 500px;
  margin: 20px auto;
  padding: 50px;
  border: solid blue 10px;
  -webkit-box-sizing: border-box;
     -moz-box-sizing: border-box;
          box-sizing: border-box;
}

* {
  -webkit-box-sizing: border-box;
     -moz-box-sizing: border-box;
          box-sizing: border-box;
}
```를

## position

### position:static
위치를 지정하지 않음.

### position:relative
`top`, `bottom`, `left`, `right`를 같이 지정하면 위치가 바뀌지만, 다른 엘리먼트들이 같이 딸려 올라오진 않는다.

### position:fixed
`top`, `bottom`, `left`, `right`를 같이 지정하면 viewport에 위치가 고정된다.
`relative`와 달리 자신의 원래 위치가 공백으로 표시 되지 않는다.
[모바일이 개판임](http://bradfrostweb.com/blog/mobile/fixed-position/)

### position:absolute

`relative`를 잡은 조상 엘리먼트 기준.


## 위치 지정 예제(position layout)
left nav를 absolute를 잡을 때는 section이 nav height보다 작으면 nav가 container밖으로 삐져나온다.

## media queries layout

- [mdn media queries](https://developer.mozilla.org/en-US/docs/Web/Guide/CSS/Media_queries)
- [정찬명 미디어쿼리 이해](http://naradesign.net/wp/2012/05/30/1823/) - **필독**
- [width versus device-width](http://www.javascriptkit.com/dhtmltutors/cssmediaqueries2.shtml)

```css
@media screen and (min-width:600px) {
  nav {
    float: left;
    width: 25%;
  }
  section {
    margin-left: 25%;
  }
}
@media screen and (max-width:599px) {
  nav li {
    display: inline;
  }
}
```

## inline-block

float 말고 inline-block쓰자. [IE6, IE7은 지원안한다.](http://blog.mozilla.org/webdev/2009/02/20/cross-browser-inline-block/)

```css
.box {
  float: left;
  width: 200px;
  height: 100px;
  margin: 1em;
}
.after-box {
  clear: left;
}


.box2 {
  display: inline-block;
  width: 200px;
  height: 100px;
  margin: 1em;
}
```

## inline-block layout

}
```
